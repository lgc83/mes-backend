# mes-backend

MES(Manufacturing Execution System) 도메인을 기반으로 *실제 현업에서 사용 가능한 백엔드 구조를 학습·구현하기 위해 제작한  

Spring Boot 기반 REST API 서버"입니다.

단순 CRUD 구현에 그치지 않고,  "인증·보안·도메인 분리·예외 처리 등 실무에서 요구되는 핵심 요소를 중심으로 설계"했습니다.

-------------------------------------------------------------------------------------------------------------------
# 기술 스택

- Java 21
- Spring Boot
- Spring Security + JWT 기반 인증
- JPA (Hibernate)
- Gradle
- MariaDB / MySQL

-------------------------------------------------------------------------------------------------------------------
# 핵심 설계 포인트

# JWT 기반 인증 / 인가 구조
- 로그인 시 Access Token 발급
- Spring Security Filter에서 JWT 검증
- 인증 실패 / 권한 오류에 대한 공통 예외 처리
- Stateless 인증 구조로 확장성 고려

# 도메인 중심 패키지 구조
- controller / service / repository 역할 분리
- 회원, 생산 오더, 판매 오더, 구매 자재 도메인 단위로 패키지 구성
- 비즈니스 로직을 Service 계층에 집중

# 실무 기준 예외 처리
- `GlobalExceptionHandler`를 통한 공통 예외 응답 처리
- 클라이언트가 상황을 명확히 인지할 수 있는 에러 구조 설계
- 인증/인가, 검증 오류에 대한 일관된 응답 포맷 제공

-------------------------------------------------------------------------------------------------------------------
## 주요 기능

# 회원 관리
- 로그인 처리
- JWT 기반 인증

# 생산 오더 관리
- 생산 지시 등록
- 생산 지시 조회

# 판매 오더 관리
- 판매 주문 등록
- 판매 주문 조회

# 보안
- Spring Security 기반 접근 제어
- 인증 사용자만 API 접근 가능

# 구매 자재 관리 (Purchase Material)
- 구매 자재 등록 / 수정 / 삭제
- 구매 자재 목록 조회 (Paging 처리)
- 거래처 및 자재 정보 관리
- 구매 요청일 / 입고 예정일 관리
- DTO 기반 요청·응답 분리 및 Validation 적용

# 프로젝트 목적

- ERP / MES 백엔드 도메인 구조 이해
- Spring Security + JWT 인증 흐름 실습
- 실무형 REST API 설계 경험
- "백엔드 신입 개발자 포트폴리오 용도"
- 실제 면접에서 인증·보안·설계 질문에 대응할 수 있도록 설계 의도를 문서화
- 생산 · 판매 · 구매 자재 흐름을 포함한 실무형 도메인 설계 경험
