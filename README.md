# mes-backend

MES(Manufacturing Execution System) 도메인을 기반으로 *실제 현업에서 사용 가능한 백엔드 구조를 학습·구현하기 위해 제작한  

Spring Boot 기반 REST API 서버입니다.

제조 현장에서 발생하는 생산·판매·구매 흐름을 시스템으로 관리한다는 전제 하에, 실제 업무 시나리오를 가정하여 API 구조를 설계했습니다.

단순 CRUD 구현에 그치지 않고,  "인증·보안·도메인 분리·예외 처리 등 실무에서 요구되는 핵심 요소를 중심으로 설계"했습니다.

특히 각 기술 요소에 대해 “왜 이 방식을 선택했는지”를 면접에서 설명할 수 있도록 설계 의도와 판단 근거를 문서화하는 데 집중했습니다.

-------------------------------------------------------------------------------------------------------------------
# 전체 아키텍처 개요

- Client → REST API 호출
- Spring Security Filter에서 JWT 인증/인가 처리
- Controller는 요청/응답 책임만 담당
- Service 계층에서 도메인 비즈니스 로직 처리
- Repository(JPA)를 통한 영속성 관리
- Global Exception Handler를 통한 공통 예외 응답 처리

-------------------------------------------------------------------------------------------------------------------
# 기술 스택

- Java 21
- Spring Boot
- Spring Security + JWT 기반 인증
- JPA (Hibernate)
- Gradle

-------------------------------------------------------------------------------------------------------------------
# 핵심 설계 포인트

# JWT 기반 인증 / 인가 구조
- 로그인 시 Access Token 발급
- Spring Security Filter에서 JWT 검증
- 인증 실패 / 권한 오류에 대한 공통 예외 처리
- Stateless 인증 구조로 확장성 고려
- 세션 기반 인증 대신 JWT를 사용하여 서버 상태 의존성을 제거하고, 향후 다중 서버 환경에서도 인증 구조를 유지할 수 있도록 설계했습니다.

# 도메인 중심 패키지 구조
- controller / service / repository 역할 분리
- 회원, 생산 오더, 판매 오더, 구매 자재 도메인 단위로 패키지 구성
- 비즈니스 로직을 Service 계층에 집중
- 기능 중심이 아닌 도메인 중심 구조를 적용하여 기능 확장 및 유지보수 시 코드 영향 범위를 최소화하도록 설계했습니다.

# 실무 기준 예외 처리
- `GlobalExceptionHandler`를 통한 공통 예외 응답 처리
- 클라이언트가 상황을 명확히 인지할 수 있는 에러 구조 설계
- 인증/인가, 검증 오류에 대한 일관된 응답 포맷 제공
- 프론트엔드 및 외부 시스템 연동 시 에러 처리 기준이 불명확하면 유지보수가 어려워지기 때문에 예외 응답 구조를 공통화했습니다.

## 주요 기능

# 정리

본 프로젝트는 “동작하는 코드”를 넘어서 왜 이런 구조를 선택했는지 설명할 수 있는 백엔드 프로젝트를 목표로 합니다.
단순 기능 구현이 아닌, 실제 실무 환경에서 요구되는 구조와 판단 과정을 이해하고 이를 설명할 수 있는 개발자가 되기 위한 프로젝트입니다.
